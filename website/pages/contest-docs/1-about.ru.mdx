# Введение

В современном мире мобильные приложения стали неотъемлимой частью нашей жизни, служа для самых
различных целей - от общения и развлечений до образования и работы. В последнее время
стали набирать популярность приложения, нацеленные на показ всевозможного медиаконтента.
Наряду с этим появилось множество сервисов-хранилищ такого контента - стоковые изображения,
видеохостинги и т. д.

Но вот в чём проблема: существующие сервисы просмотра медиаконтента имеют сложный и непонятный
интерфейс пользователя (UI)[^1], зачастую представленный в виде REST[^2] API[^3], который может
использовать только разработчик с соответствующим навыком.

Целью данного проекта является разработка мобильного приложения, упрощающего доступ к сервисам просмотра медиаконтента, конкретно - изображений.
Акцент в работе будет поставлен на возможность задания собственных источников контента.

**Задачи**:
1. Выбрать ПО для разработки мобильного приложения
2. Разработать функционал (back-end) приложения
3. Разработать оформление (front-end) приложения
4. Протестировать приложение
5. Написать сопровождающий веб-сайт и документацию



[^1]: User Interface (UI, пользовательский интерфейс) - интерфейс, обеспечивающий передачу информации между
пользователем (человеком) и программно-аппаратными компонентами компьютерной системы (некой программой).
Зачастую данный термин употребляется по отношению к графическому интерфейсу пользователя (GUI).
[^2]: Representational State Transfer (REST, буквально - передача "самоописываемого" состояния) -
стиль архитектуры программного обеспечения для распределенных систем, в котором веб-сервис представляет
собой совокупность веб-ресурсов, доступ к которым осуществляется
посредством обмена сообщениями между клиентом и сервером.
[^3]: Application Programming Interface (API) - набор соглашений о программировании - интерфейсов, которые позволяют
программам взаимодействовать друг с другом.
